Q 01

Design Half Adder (HA) & Full Adder (FA) with Logic Diagrams

Half Adder (HA)

A Half Adder adds two single-bit binary numbers (A, B) and produces a Sum and a Carry. It cannot handle an incoming carry.

Sum = A ⊕ B   |   Carry = A · B
Half Adder — Logic Diagram

Truth Table — Half Adder

Working

XOR gate computes the Sum: output is 1 only when inputs differ.

AND gate computes the Carry: output is 1 only when both inputs are 1.

Limitation: Cannot accept a carry-in bit — hence called "Half" adder.

Full Adder (FA)

A Full Adder adds three bits: A, B, and a Carry-In (Cin). It produces a Sum and Carry-Out (Cout).

Sum = A ⊕ B ⊕ Cin   |   Cout = AB + Cin(A ⊕ B)

Construction

HA₁: Adds A and B → gives intermediate Sum S₁ and Carry C₁.

HA₂: Adds S₁ with Cin → gives final Sum and Carry C₂.

OR gate: Combines C₁ and C₂ to produce the final Cout.

Advantage over HA: Handles carry-in, so full adders can be cascaded to build multi-bit adders.

Q 03

Construct a D Latch and Explain

A D Latch (Data/Delay Latch) is built from an SR Latch with an Enable (E) input. The D input connects to S, and D' connects to R, preventing the invalid S=R=1 condition.

When E=1: Q follows D  |  When E=0: Q holds its last value
D Latch — NAND Gate Implementation

Truth Table — D Latch

EDQ (next)Q' (next)
0XQ (hold)Q' (hold)
1001
1110

Working Principle

The D latch eliminates the forbidden state (S=R=1) by ensuring S and R are always complements:

  • When E=0: Both NAND outputs = 1 → latch retains state (memory mode)
  • When E=1, D=1: S=1, R=0 → Q=1 (Set)
  • When E=1, D=0: S=0, R=1 → Q=0 (Reset)

The dashed lines represent feedback paths that maintain state.

Q 04

Design a 4-bit Adder-Subtractor Circuit & Explain

A 4-bit Adder-Subtractor uses a control signal M. When M=0, it adds; when M=1, it subtracts (using 2's complement: A − B = A + B̄ + 1).

M=0: S = A + B (Addition)   |   M=1: S = A − B = A + B̄ + 1 (Subtraction)
4-bit Adder-Subtractor using XOR gates + Full Adders

Operation Table

MOperationResult
0AdditionS = A + B
1SubtractionS = A − B (2's complement)

Working

  • Each XOR gate receives Bᵢ and M. When M=0, XOR passes Bᵢ unchanged. When M=1, XOR inverts Bᵢ (giving B̄ᵢ).
  • M is also fed as Cin to FA₀. So when M=1, Cin=1 completes the 2's complement (B̄+1 = −B).
  • All 4 FAs are cascaded — carry ripples from FA₀ to FA₃.
  • The final Cout indicates overflow in addition or borrow in subtraction.

Q 13

Latch, Counter & Ripple Counters

Latch

A Latch is a level-triggered bistable memory element. It stores one bit of data and changes state as long as the enable/clock signal is HIGH.

  • SR Latch, D Latch, JK Latch
  • Responds to input level (not edge)
  • Used in asynchronous design and simple storage

Counter

A Counter is a sequential circuit that goes through a predefined sequence of states upon application of clock pulses. Categories:

SynchronousAsynchronous (Ripple)
  • Up Counter: counts 0→1→2→…→N
  • Down Counter: counts N→…→1→0
  • Up-Down Counter: direction controlled by signal
  • Modulo-N Counter: resets after N counts

Modulus of a Counter

Modulus = Number of valid states = 2ⁿ (for n FFs in natural binary)

A 4-bit counter has Mod = 16 (counts 0–15).

Ripple Counter (Asynchronous Counter)

In a Ripple Counter, the output of each FF serves as the clock for the next FF. The clock signal "ripples" through the chain — hence the name.

Each FF divides the clock by 2: f_out = f_clk / 2ⁿ
4-bit Ripple Counter (JK FFs, J=K=1)

Ripple Counter Features

  • Asynchronous — each FF clocked by Q̄ of previous
  • Simple design, few connections
  • Propagation delay accumulates (ripple delay)
  • Not suitable for high-speed applications
  • Output Q₀ is LSB (fastest toggling)

Ripple Counter Timing

Q₀ toggles every 1 clock pulse
Q₁ toggles every 2 clock pulses
Q₂ toggles every 4 clock pulses
Q₃ toggles every 8 clock pulses
